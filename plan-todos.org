** What

New "Anki Flashcards" sidebar panel with two dedicated buttons: [Basic]

Preview raw output → accept saves as =.basic.txt= / =.cloze.txt= under

** Key decisions
:PROPERTIES:
:CUSTOM_ID: key-decisions
:END:
- Two independent buttons instead of one mixed request: user controls
  type, no partial-merge complexity
- Separate =.basic.txt= / =.cloze.txt= files so Anki import maps each
  file to correct note type (mixed types in one import file not
  supported by Anki)
- Cloze format is single-line (no tabs): =sentence with {{c1::blank}}=
  --- Anki's Cloze note type has only one text field
- =_generationId= counter discards stale responses if user clicks a new
  generation before previous one completes
- Each prompt says "Cover EVERY section --- do not skip any part" to
  prevent lazy/partial generation
- Flashcards path defaults to =$HOME/magnus/nextlearn/flashcards= (same
  pattern as DecksPath: configurable in Settings with Browse button)
- File naming: =<deck-filename>.basic.txt= and
  =<deck-filename>.cloze.txt=

*** Render LaTeX, code blocks, blockquotes in quiz WebView
:PROPERTIES:
:CUSTOM_ID: render-latex-code-blocks-blockquotes-in-quiz-webview
:END:
​****What****

Question/explanation text containing LaTeX ($...$ /

#+begin_src math
...
#+end_src

), code fences (```), or blockquotes (>) currently renders as raw text
instead of styled HTML. The quiz WebView lacks KaTeX, highlight.js, and
block-level CSS.

​****Root cause****

- =McqQuizHtmlBuilder= pipes all text through
  =MarkdownInlineRenderer.RenderInline= which only handles inline
  constructs (bold, italic, code spans, inline math)
- The quiz HTML template has no KaTeX CSS/JS injection --- math
  delimiters show as raw =$x^2$= instead of rendered KaTeX
- No highlight.js → code fences displayed as literal =~= ``==~` text
- No CSS for =<pre>=, =<blockquote>=, =<table>= --- even if HTML
  existed, it would be unstyled
- The main ContentWebView solves all this via =WebViewBridge.LoadHtml= +
  =HtmlContentBuilder.Build()=, but the quiz WebView bypasses both

​****Changes****

*** 1. HtmlContentBuilder.cs --- add static =RenderBlock(string text)= method
:PROPERTIES:
:CUSTOM_ID: htmlcontentbuilder.cs-add-static-renderblockstring-text-method
:END:
Extract the block-level processing loop from =Build()= into a new static
method:

~public static string RenderBlock(string text, bool isOrgFile = false,
string? imageDir = null, List? imagePaths = null,
IReadOnlyDictionary<string, string>? footnotes = null, string? decksPath
= null, string? currentDir = null)~

This splits input into lines, runs through the same handler chain as
=Build()= (code fences → blockquotes → tables → lists → headings → math
blocks → inline), and returns the body HTML without =WrapInHtml=
wrapper. =Build()= delegates to it internally.

*** 2. WebViewBridge.cs --- extract =EnrichHtml= static helper
:PROPERTIES:
:CUSTOM_ID: webviewbridge.cs-extract-enrichhtml-static-helper
:END:
Move the KaTeX/highlight.js/key-forwarding injection from =LoadHtml=
into:

=public static string EnrichHtml(string html)=

Uses =AppContext.BaseDirectory= to locate =Assets/katex/= and
=Assets/atom-one-dark.min.css= / =Assets/custom-highlight.js=. Replaces
=<!--KATEX_CSS-->=, =<!--KATEX_CSS-->=, =<!--HIGHLIGHT_CSS-->=,
=/* HIGHLIGHT_JS */= placeholders.

=WebViewBridge.LoadHtml= calls =EnrichHtml(html)= then sets Source.

*** 3. McqQuizHtmlBuilder.cs --- add block-level CSS + rendering
:PROPERTIES:
:CUSTOM_ID: mcqquizhtmlbuilder.cs-add-block-level-css-rendering
